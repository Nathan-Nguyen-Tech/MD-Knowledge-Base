## ‚úÖ POST-GENERATION VALIDATION

After generating each card, verify:

### 1. COMPLETENESS CHECK

**Patient-Facing Side**:
- [ ] WHAT YOU'RE DOING (1 sentence)
- [ ] WHY THIS MATTERS TO YOU (personal goal)
- [ ] HOW TO DO IT (3-5 bullet points)
- [ ] WHEN TO DO IT (frequency + timeline)
- [ ] WHAT TO WATCH FOR (2-3 red flags)

**Full-View Side**:
- [ ] WHAT (complete details)
- [ ] WHO (target population + considerations)
- [ ] WHEN (frequency, duration, timing)
- [ ] WHERE (setting)
- [ ] WHY (clinical goal + mechanism + evidence)
- [ ] HOW (detailed protocol)
- [ ] MEASURE (tracking method + tools + frequency)
- [ ] TARGET (goal + timeline + success criteria)
- [ ] SAFETY (by health systems + contact guidelines)
- [ ] FOLLOW-UP (specific checkpoints)
- [ ] NOTES (barriers + tips + interactions)

**Metadata**:
- [ ] Tags (minimum 3, maximum 10)
- [ ] Evidence grade (A/B/C)
- [ ] Feasibility score (0-100%)
- [ ] Version number
- [ ] Last updated date

### 2. QUALITY CHECK

**Reading Level** (Patient-Facing Only):
- [ ] 6th-8th grade reading level
- [ ] Sentences <15 words (most of them)
- [ ] Active voice used
- [ ] Common words (not jargon)
- [ ] Medical terms defined when necessary

**Quick Test**: Can a middle schooler understand the patient-facing side?

**Clinical Accuracy**:
- [ ] Dosing includes unit and route (medications)
- [ ] Timing includes frequency
- [ ] Safety section includes emergency criteria
- [ ] Evidence citation included with grade
- [ ] Monitoring requirements specified

**Performance Medicine Alignment**:
- [ ] Clinical goal mapped to 8 Health Systems
- [ ] Domain specified (Structure/Function/Risk)
- [ ] Example: "Cardiovascular System: Reduce Risk"

### 3. SAFETY VERIFICATION

- [ ] Safety section organized by health systems
- [ ] "When to contact provider" has 3 tiers:
  - Routine (can wait for scheduled visit)
  - Urgent (call within 24 hours)
  - Emergency (call 911 immediately)
- [ ] Top 3 red flags in patient-facing view
- [ ] Contraindications listed (absolute and relative)
- [ ] Drug interactions noted (if medication)
- [ ] Food interactions noted (if applicable)

### 4. ACTIONABILITY CHECK

- [ ] Card specifies ONE clear action
- [ ] Action is measurable
- [ ] Timeline for results is realistic
- [ ] Tools needed are accessible
- [ ] Instructions are step-by-step

**Test**: Could a patient execute this card without additional questions?

### 5. TAG VALIDATION

**Required Tag Categories** (select at least one from each):
- [ ] Clinical Intent (#PreventiveCare, #ChronicManagement, etc.)
- [ ] Care Setting (#Home, #Outpatient, etc.)
- [ ] Organ System (#Cardiology, #Metabolic, etc.)

**Optional But Recommended**:
- [ ] Guidelines (#USPSTF_GradeA, #ACC_AHA, etc.)
- [ ] Life Stage (#Adult, #Geriatric, etc.)
- [ ] Condition (#Hypertension, #Diabetes, etc.)

### FEASIBILITY SCORING GUIDE (0-100%)

**Base Score**: 100%

**Deduction Guidelines**:

| Barrier Type | Deduction | Example |
|--------------|-----------|---------|
| No insurance coverage | -25% | Expensive imaging not covered |
| High cost (>$100/month) | -20% | Brand-name only medication |
| Equipment not accessible | -15% | Home dialysis machine |
| Transportation barriers | -15% | Weekly PT in distant location |
| Time intensive (>30 min/day) | -10% | Complex meal prep |
| Complex instructions (>5 steps) | -10% | Multi-step wound care |
| Language barriers | -15% | No translation available |
| Literacy mismatch | -10% | College-level materials |
| Previous failure | -20% | Patient tried and couldn't sustain |
| Contraindication/allergy | -100% | Card not offered |

**Booster Guidelines**:

| Facilitator | Addition | Example |
|-------------|----------|---------|
| Previous similar success | +15% | Successfully used similar med |
| Patient specifically requested | +20% | Patient motivated, asked for this |
| Low time burden (<10 min/day) | +10% | Simple daily pill |
| Free or fully covered | +10% | Generic medication, preventive screening |
| Can do at home | +10% | Home exercises vs gym |
| Strong social support | +10% | Family committed to helping |
| High community rating (>4‚òÖ) | +5% | Other patients loved it |

**Score Interpretation**:
- **90-100%**: Highly feasible - recommend strongly
- **70-89%**: Feasible with support - discuss barriers
- **50-69%**: Challenging - need barrier mitigation plan
- **<50%**: Low feasibility - strongly consider alternatives

### EVIDENCE GRADE QUICK REFERENCE

**Grade A** - Use when:
- Multiple high-quality RCTs
- Consistent findings across studies
- Strong guideline recommendation (Class 1)
- Examples: Statin for CVD prevention, Metformin for T2DM, Aspirin for MI prevention

**Grade B** - Use when:
- Limited RCTs or some conflicting data
- Moderate-quality evidence
- Guideline Class 2a recommendation
- Examples: Omega-3 for heart health, Specific diet patterns, Some screening tests

**Grade C** - Use when:
- Observational studies or case series
- Expert opinion / consensus
- Guideline Class 2b-3 recommendation
- Examples: Some supplements, Emerging therapies, Practice-based recommendations

**Common Guideline Sources**:
- **USPSTF**: Preventive services (screening, counseling)
- **ACC/AHA**: Cardiology (BP, cholesterol, heart failure)
- **ADA**: Diabetes (glucose management, screening)
- **KDIGO**: Kidney disease
- **ATS/ERS**: Pulmonary (asthma, COPD)
- **NICE**: UK guidelines (internationally recognized)
- **Cochrane**: Systematic reviews (gold standard evidence)
- **ACOG**: Obstetrics and gynecology
- **AAP**: Pediatrics

### CLINICAL GOAL FORMATTER

**Format**: [System] System: [Action Verb] [Domain]

**8 Systems**:
1. Neurological
2. Cardiovascular
3. Respiratory
4. Metabolic
5. Filtration/Urinary
6. Musculoskeletal
7. Immune
8. Reproduction

**3 Domains**:
- Structure (anatomy, tissue integrity)
- Function (physiological performance)
- Risk (disease prevention)

**Action Verbs by Domain**:
- Structure: Restore, Repair, Strengthen, Protect
- Function: Improve, Optimize, Enhance, Maintain
- Risk: Reduce, Prevent, Minimize, Mitigate

**Examples**:
- "Cardiovascular System: Reduce Risk" (statin for cholesterol)
- "Metabolic System: Improve Function" (metformin for glucose control)
- "Musculoskeletal System: Restore Structure" (calcium for bone density)
- "Neurological System: Improve Function" (SSRI for depression)
- "Respiratory System: Maintain Function" (inhaler for asthma control)
- "Immune System: Reduce Risk" (vaccination for infection prevention)

### BARRIER CHECKLIST TEMPLATE

**Every card should address**:

- [ ] **Cost**: Approximate monthly cost, generic availability, insurance coverage
- [ ] **Time**: Daily time commitment
- [ ] **Access**: Where to get it, prescription needed?
- [ ] **Physical Ability**: Any physical requirements or limitations
- [ ] **Knowledge**: Education level needed
- [ ] **Social Support**: Need for family/caregiver help
- [ ] **Transportation**: Pharmacy, lab, clinic visits required
- [ ] **Language**: Available in other languages?
- [ ] **Equipment**: Special devices or supplies needed

**Pro tip**: The more barriers identified, the more realistic the feasibility score.

### COMMON SAFETY WARNINGS BY CARD TYPE

**Medication Cards - Always Include**:
- Common side effects (top 5 with percentages)
- Serious/emergent reactions (when to call 911)
- Drug-drug interactions (major ones)
- Special population warnings (pregnancy, breastfeeding, renal/hepatic)
- Contraindications (absolute and relative)

**Nutrition Cards - Always Include**:
- Allergy warnings
- Blood sugar impact (if diabetic)
- Medication-food interactions
- Hydration requirements
- Dietary restrictions (religious, cultural)

**Exercise Cards - Always Include**:
- Contraindications (chest pain, severe shortness of breath)
- When to stop (warning signs)
- Injury prevention
- Progression safety
- Equipment safety

**Lab/Test Cards - Always Include**:
- Preparation requirements (fasting, medication holds)
- Normal vs abnormal results interpretation
- Follow-up plan based on results
- Critical values requiring immediate action

## üìä VERSION CONTROL STANDARDS

Every card should track:
- **Version number**: v[Major].[Minor]
- **Last updated**: [Date]
- **Next review**: [Date]
- **Changelog**: List of what changed in each version

**When to increment version**:
- **Major version** (v1.0 ‚Üí v2.0): Significant changes (new evidence, dosing changes, structure changes)
- **Minor version** (v1.0 ‚Üí v1.1): Small updates (typo fixes, clarifications, formatting)

**Example Changelog**:
```
v2.1 (Oct 2024): Updated dosing per new ACC/AHA 2024 guidelines; added renal dosing table
v2.0 (Jan 2024): Added community feedback section; improved patient-facing language
v1.5 (June 2023): Initial release
```

## üîÑ QUARTERLY REVIEW PROCESS

**AI Agent Responsibilities**:
1. Scan medical literature for new evidence on existing cards
2. Flag cards when:
   - New guidelines published
   - Contradictory evidence emerges
   - Drug safety alerts issued (FDA/WHO)
   - Dosing recommendations change
3. Notify clinical team of flagged cards

**Clinical Team Responsibilities**:
1. Review flagged cards
2. Update content with new evidence
3. Increment version number
4. Document changes in changelog
5. Notify patients on outdated cards

## üö® CRITICAL ERROR CHECKS

**Never publish a card that**:
- Uses >8th grade reading level in patient view
- Omits dosage units for medications
- Lacks emergency criteria in safety section
- Has no evidence citation or grade
- Contains medical jargon unexplained
- Is missing any required template sections
- Has contradictory information between views
- Shows feasibility score without calculation basis
