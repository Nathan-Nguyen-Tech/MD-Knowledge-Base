# CARE PLAN DIGITAL INTERFACE SPECIFICATION

**Version:** 0.1
**Date:** 2025-12-13
**Status:** Pre-release (v1.0 = first production deployment)
**Scope:** Patient-Facing Care Plan Display System

---

## PURPOSE

This specification defines **how the Care Plan is displayed to patients** as a digital interface. The Care Plan is a **navigation layer** that organizes SMART Cards by time interval - it is NOT a document containing full card content.

**Core Principle:**
> The Care Plan is a **clickable, scannable list view** organized by time intervals. SMART Cards are interactive elements that expand on tap/click to reveal detail. The Care Plan itself stays clean and lightweight.

---

## DESIGN PHILOSOPHY

### What the Care Plan IS
- A **navigation interface** to your SMART Cards
- **Organized by time interval** (Daily â†’ Weekly â†’ Monthly â†’ Quarterly â†’ Annually)
- **Scannable** - see everything at a glance
- **Interactive** - tap/click to expand card details
- **Contextual** - each time interval shows relevant guidance

### What the Care Plan IS NOT
- A document containing full SMART Card content
- A static PDF (though exportable on demand)
- A replacement for individual card views
- Overwhelming or cluttered

## TIME INTERVAL STRUCTURE

### Available Intervals

| Interval | Display Frequency | Purpose | Typical Card Types |
|----------|-------------------|---------|-------------------|
| **Daily** | Every day | Routine actions | Medications, micro-habits, daily movement |
| **Weekly** | Specific days | Structured activities | Strength training, meal prep, therapy sessions |
| **Monthly** | Once per month | Monitoring & check-ins | Weight tracking, BP review, symptom assessment |
| **Quarterly** | Every 3 months | Assessments & adjustments | Labs, care plan review, milestone check |
| **Semi-Annually** | Every 6 months | Intermediate reviews | Specialist follow-ups, imaging |
| **Annually** | Once per year | Comprehensive | Full assessment, screenings, plan renewal |
| **As Needed (PRN)** | When triggered | Acute response | PRN medications, symptom cards, crisis protocols |

## CONTEXTUAL GUIDANCE BY INTERVAL

Each time interval should include contextual information to help users understand what they're looking at.

### Daily Context Elements
- Time-of-day greeting
- Today's focus/priority
- Yesterday's completion rate
- Tomorrow preview
- Streak/momentum indicators

### Weekly Context Elements
- Week theme/focus
- Day-by-day schedule
- Weekly targets with progress
- Appointment reminders
- Week-over-week comparison

### Monthly Context Elements
- Month theme
- Trend visualizations (weight, BP, etc.)
- Appointment calendar
- Adherence summary
- Medication refill alerts

### Quarterly Context Elements
- Quarter theme
- Milestone checklist
- Lab results with trends
- Outcome metrics
- Next quarter preview + unlock status

### Annual Context Elements
- Year goal
- Health Matrix overview
- Quarter-by-quarter summary
- Year-end outcomes
- Next year planning prompt
